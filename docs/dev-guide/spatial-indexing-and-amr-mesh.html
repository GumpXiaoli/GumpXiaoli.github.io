CAE Fundamentals - Spatial Indexing and AMR Mesh

Background​

Efficient spatial indexing is important for CAE.

信息
  • CAE computation needs spatial discretization on PDE.
  • Some spatial query operations (neighbour search, distance calculation, etc.) are needed.
  • Efficient spatial indexing and queries are critical for domain decomposition and parallelism.

AMR is extremely suitable for complicated problems

信息
  • Most complicated problems are of multi-physics and multi-scale (both time and space).
  • Adaptivity is often need to reduce footprint and computation cost.

Spatial Indexing​

What is Spatial Indexing?​

Spatial indices are used by spatial databases to optimize spatial queries. Spatial databases are databases which store information related to objects in space.

Suitable Spatial Indexing Methods should handle spatial queries efficiently. The queries should allow for the use of geometry data types (such as points, lines and polygons) and consider the spatial relationship between geometries.

How to Define a Good Spatial Indexing Method?​

Suitable Spatial Indexing Methods should handle spatial queries efficiently.

The queries should allow for the use of geometry data types (such as points, lines and polygons) and consider the spatial relationship between geometries.

Typical queries includes:

  • Neighbouring Search
  • Distance Calculation
  • Overlaps Detection
  • Area and Length Calculation

R-tree​

  • First proposed by Guttman in 1984.
  • Data Layout: Minimum Bounding Box based Tree
  • One box can contains multi-objects.
  • Improvements: R*-tree, ...

KD-Tree​

  • Very suitable for multi-dimensional point clouds.
  • Using binary-cutting to construct the tree.

Oct-tree​

  • Level of details in 3D rendering.
  • Local refinement/adaptivity easily and efficiently.
  • Can be implemented without geometric overlap.

Space Filling Curve​

  • Mapping from multi-dimension to one dimension;
  • Preserve data locality;

AMR (Adaptive Mesh Refinement)​

Why CAE community need AMR Mesh?​

Local refinement for PDE is always needed for complicated problems.

  • Multi-scale in many aspects (time, space, gradients, etc.). (Complicated)
Examples
  • Shocks and interfaces.
  • Self-gravitating flows in astrophysics.
  • Complex engineering geometries.
  • Combustion.
  • Magnetohydrodynamics: space weather, magnetic fusion.
  • We only have limited resources. (Affordable)

One adjusts the computational effort locally to maintain a uniform level of accuracy throughout the problem domain.

What AMR architecture should offer?​

  • Local refinement as desired.
  • Local coarsening as desired.
  • Dynamic adaptivity of mesh architecture.
  • Parallel partition between processes.
Notes for Parallelism
  • Workload balance.
  • Minimize the communication.
  • Minimize the difference from current partion.
  • Fast local neighbour search (both local and remote).

Types of AMR Mesh​

Conforming Unstructured AMR​

Distorted Structured Mesh (Conforming)​

  • Mesh quality is still a problem.
  • Unavoidable memory waste in most cases.
  • Still hard to handle complex geometries.

Octree-based AMR (Non-Conforming)​

  • Non-overlap Mesh (Only leaves are stored in memory)
  • Fast elementary operations
NOTES for Elementary Operations
  • Refine or coarsening locally.
  • Bit-operations from index to geometric coordinates.
  • Fast neighbour (in-processor or off-processor) O(log *)

Block-structured AMR (Non-Conforming)​

  • The domain is composed of a union of rectangular regions.
  • Overlap between levels and only fine levels should be cared.
  • Connectivity is simple and vectorization is easy.
  • Intuitive parallelism and good use of memory cache.
  • Local copy is unavoidable and inefficient to some extent.

Some Good Open-Source AMR Projects​

AMReX​

  • Block-Structured AMR; (formerly known as BoxLib)
  • Base for many different application codes;
  • MPI/OPENMP, GPU (both CUDA and HIP), DPC++ parallelization support;
  • Original base framework in Supreium;

p4est​

  • Forests of Octree AMR;
  • Geometric flexibility;
  • No need to construct cubic meshes; (Only structured mesh)
  • Only mesh interface; (deal.ii, a FEM library using p4est)

PARAMESH​

  • Block-structured AMR with tree-based Block;
  • All written in Fortran;
  • A subset of AMReX to some extent in capabilities;

SAMRAI​

  • Block-structured AMR;
  • Mesh architecture is only a local view;
  • Some key operations (clustering, partition, etc) are distributed;
